## 2. Stack Decisions

| Layer | Choice | Why |
|-------|--------|-----|
| **Execution substrate** | Cowork Schedule task on local Mac | Removes the API turn timeout that was killing the v1.7 Routine. Local sessions can run 12+ minutes without ceremony. |
| **GitHub auth** | Local `gh` CLI with fine-grained PAT | Single keychain entry on the Mac. Revocable from GitHub UI if Mac is lost. No server-side credential to rotate. |
| **Push model** | Direct to `main`, no PR | User is the only committer. PR + auto-merge added latency and a remote-validator failure surface for zero collaboration benefit. |
| **Validator** | Local TypeScript script `scripts/validate-brief.ts` invoked via `npx tsx` | Production script — was already running inside the v1.7 GitHub Action. v1.8 just changes the invocation site to the local Cowork session. Removes the remote-validation timeout/quota failure mode. |
| **Site framework** | Next.js 14+ (App Router, static export) | Already in production. Not changed. |
| **Hosting** | Vercel | Already in production. Auto-deploy webhook on push to `main`. Not changed. |
| **Package manager** | `npm` | Already in production. |
| **Notifications** | `osascript -e 'display notification ...'` (macOS native) | Zero install. Replaces email-from-GitHub-Action. Optional upgrade path: `terminal-notifier` for clickable links. |
| **Wake management** | `pmset -c sleep 0` (always-awake on AC) | Mac mini configuration; simplest. Laptop users use scheduled wake events instead. |

**Explicitly retired in v1.8 (Claude Code should remove these):**
- Anthropic-managed Claude Routine at `claude.ai/code/routines`
- Claude GitHub App installation on the repo
- `.github/workflows/validate-brief.yml`
- `auto_merge` repo setting
- Branch protection requiring "validate" status check

## 4. Data Schemas (Preserved from v1.7 — Reference Only)

The brief data model does not change in v1.8. This section is for Claude Code to verify, not modify.

### 4.1 Brief MDX file — `content/briefs/YYYY-MM-DD-day-NNN.mdx`

YAML frontmatter:

```yaml
---
day: 59                              # integer; today − Feb 28, 2026
date: 2026-04-27                     # YYYY-MM-DD; must match filename
ceasefire_day: 19                    # integer or null; ceasefire started Apr 8
headline: "One-line summary, ~10–15 words"
sources_count: 14                    # integer; ≥ 8 unless quiet_day
quiet_day: false                     # boolean
retroactive: false                   # boolean; true if brief is being filled in for a missed day
reading_time_minutes: 12             # integer; informational
---
```

Body sections (each `# Heading`, in order):
1. Executive Summary (150–400 words)
2. Top Developments (4–5 numbered)
3. Analytical Judgment (400–600 words)
4. Scenarios and Probabilities (3–5)
5. Casualties (markdown table)
6. Taiwan Angle (200–300 words)
7. Sources (bulleted URLs)

Optional appended subsection (evening run only):
- Evening flash notes

### 4.2 Brief data file — `content/briefs/YYYY-MM-DD-day-NNN.data.ts`

TypeScript module exporting a `BriefData` object. Schema defined in `lib/brief-data.ts`:

```typescript
export type BriefData = {
  escalation: EscalationGaugeProps;
  events: EventRow[];
  casualties: CasualtiesTableProps;
  exec?: string;                     // Executive summary as plain text
  implications?: Implication[];      // Strategic implications array
  flash?: string;                    // Evening flash notes (added by evening run)
  casualtyNotes?: ActorCasualtyNotes;
};
```

Validator enforces:
- `escalation.direction` ∈ {escalating, mixed, deescalating}
- `escalation.risk7d` ∈ {extreme, high, medium, low}
- `escalation.spillover` ∈ {expanding, conditional, contained}
- `events[]`: every entry has `id`, `direction`, `importance`, `source`, `event`, `summary`, `impact`
- `casualties`: required actor breakdown (US / Israel / Iran / Other)

### 4.3 Rolling context — `content/context.md`

Plain markdown, ~600–1000 words. Sections:
1. Current war status
2. Cumulative state (casualties, energy, interceptors)
3. Active deadlines
4. Diplomatic state
5. Standing analytical threads
6. What to watch tomorrow
7. Evening flash notes (one block per day, freshest at top, monthly audit prunes oldest)

### 4.4 Brief registry — `lib/brief-data.ts`

Pattern (existing — append a line per new day):

```typescript
import day059 from '@/content/briefs/2026-04-27-day-059.data';

const briefDataBySlug: Record<string, BriefData> = {
  // ... existing days ...
  '2026-04-27-day-059': day059,
};
```

## 7. Build Scripts & Prebuild Hooks

### 7.1 `scripts/validate-brief.ts` — Local Validator (CRITICAL — already exists in production, verify behaviour)

Production TypeScript script. Invoked by the Cowork Schedule task as `npx tsx scripts/validate-brief.ts content/briefs/<file>.mdx` before any commit. Same script that previously ran inside `.github/workflows/validate-brief.yml`; v1.8 just moves the invocation site to the local Cowork session.

Checks (must all pass for the skill to commit):

1. **Schema** — gauges, probabilities, casualties shape per §4.2
2. **URL resolution** — HEAD (fall back to GET on 405/403) on every URL in `## Sources`, 8s timeout. URLs containing `web.archive.org` short-circuit to OK (no network call). **Fails if more than 2 URLs return non-OK.** ≤ 2 broken bare URLs is a warning only.
3. **Executive Summary word count** — extracted from `## Executive Summary`, must be in **[150, 300]**.
4. **Strategic Implications word count** — entire `## Strategic Implications` section, all subsections combined, must be in **[400, 800]**.
5. **Total body word count** — `body + sidecarProse(data)` must be in **[1200, 4000]**. `sidecarProse` concatenates `escalation.rationale.{direction,risk7d,spillover}`, every event's `event/summary/impact/source`, and every actor's `casualties.cumulative/status`.
6. **Date consistency** — frontmatter date matches filename; day number = prior brief + 1
7. **Research depth** — `sources_count` ≥ 8 unless `quiet_day: true`
8. **Source diversity** — `source_categories` ≥ 6 distinct values

`gh auth status` is called by the skill itself as a separate pre-push step, not by the validator script.

Exit codes:
- `0` — all checks pass; skill proceeds to commit + push
- `1` — one or more checks fail; skill writes the validator's stdout to `validator-report.txt` and aborts

**The skill is responsible for never handing this validator a brief that should fail.** Per §3.5.12 (URL wrapping) and §3.5.13 (pre-emit word counts) of `DESIGN_v1.8.md`, the skill performs equivalent checks at draft time. The validator is a backstop for drafting bugs and infrastructure regressions, not the primary length governor.

### 7.2 `scripts/normalize-sources.ts` — Optional URL Archive-Wrap Transform (NEW v1.8)

Optional helper that rewrites every URL in an MDX file's `## Sources` section to its `web.archive.org/web/2026/<original>` form. Idempotent. Per §3.5.12 of `DESIGN_v1.8.md`, the skill should perform this transform itself before write — this script is a backup utility for manual fixes (e.g., a needs-review brief with URL-resolution failures).

```bash
# Pipe form
npx tsx scripts/normalize-sources.ts < content/briefs/2026-04-27-day-059.mdx > /tmp/normalized.mdx

# In-place form
npx tsx scripts/normalize-sources.ts --in-place content/briefs/2026-04-27-day-059.mdx
```

Full implementation in Appendix C.8 of `DESIGN_v1.8.md`. Claude Code creates this file as part of Phase 3.
